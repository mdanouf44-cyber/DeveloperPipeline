This content doctrine defines the positioning, topic selection filters, and styling parameters for all Mohammad Anouf Saani LinkedIn content. It overrides all previous or conflicting guidelines.

## 🎯 Positioning: "The Technical Builder & Architect"

We write for **developers, software engineers, systems architects, and technical builders**—people who build production-grade software, orchestrate AI pipelines, deploy local infrastructure, and want to stay ahead of the high-signal tech curve.
* **Core Lens:** Hands-on technical value, architectural breakdowns, open-source leverage, and ecosystem shifts.
* **Tone:** Authoritative, builder-credible, highly technical, actionable, and educational.
* **Brand Alignment:** `Mohammad Anouf Saani (www.linkedin.com/in/mohammad-anouf-saani)` remains the signed brand. The voice is a seasoned systems architect or developer advocate detailing engineering patterns and breakthroughs from the front lines.

---

## 🔍 The 4-Part Topic Filter

Every story, news item, repository, or dataset must pass these four filters:

1. **Tech Value:** Does this offer real engineering substance? (e.g., architectural diagrams, CLI configs, code patterns, benchmarks, vector search strategies—not generic business fluff).
2. **Actionability:** Can a developer run, install, or implement this within 24 hours? (e.g., an Ollama run command, Docker Compose setup, or a CrewAI task orchestration config).
3. **Educational Depth:** Does it explain *how* something works under the hood (e.g., chunking strategies, embeddings, or MCP protocol handoffs) rather than just saying it exists?
4. **Ecosystem Signal:** Does it capture a high-signal trend or debate from places like Hacker News, OpenAI Developer Forums, or rising GitHub repositories?

---

## ❌ The DROP List (Banned Topics)

Never write about or include:
* **Generic Hype & Marketing Fluff:** Vague claims about "AI changing the world" or "10x productivity" without code, architecture, or data.
* **Non-Technical Listicles:** Lists of "10 AI tools you must try" that are just simple wrapper apps.
* **Corporate / HR Brag Posts:** VC funding announcements, MRR milestones, hiring updates, or typical LinkedIn corporate/agency metrics.
* **Dry News Relay:** Plain news reporting (e.g. "OpenAI launched X") with no technical explanation of the architecture, features, or developer impact.
